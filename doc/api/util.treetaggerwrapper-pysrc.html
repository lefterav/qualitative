  88  Encoding
 
  89  ~~~~~~~~
 
  90  
 
  91  By default files and `str` strings are considered to be using latin1 encoding.
 
  92  
 
  93  - You can specify the encoding when using the script as a command-line tool,
 
  94     with the :option:`-e` param (see online help).
 
  95  
 
  96  - When using the script as an embedded tool into a larger project, you can use
 
  97    different parameter types as :meth:`TreeTagger.TagText` `text` parameter.
 
  98  
 
  99    - You can use unicode strings, and :meth:`TagText` will return list of unicode
 
 100      strings. If you fill'in TagText with a list of strings and dont provide
 
 101      and `encoding` parameter, TagText will also return unicode strings
 
 102      (same if you set `encoding` to `unicode` or `"unicode"`.
 
 103    - You can use `str` strings and specify an alternate `encoding` (and an alternate
 
 104      `errors` detection), and TagText() will automatically convert input from this
 
 105      encoding and output to this encoding.

 107  
 
 108  This module does two main things
 
 109  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
 
 110  
 
 111  - Manage preprocessing of text in place of external Perl scripts as in
 
 112    base TreeTagger installation, thus avoid starting Perl each time a chunk
 
 113    of text must be tagged.
 
 114  - Keep alive a pipe connected to TreeTagger process, and use that pipe
 
 115    to send data and retrieve tags, thus avoid starting TreeTagger each
 
 116    time.
 
 117  
 
 118  Use of pipes avoid writing/reading temporary files on disk too.
 
 119  
 
 120  Other things done by this module
 
 121  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
 
 122  
 
 123  - Can number lines into XML tags (to identify lines after TreeTagger
 
 124    processing).
 
 125  - Can mark whitespaces with XML tags.
 
 126  - By default replace non-talk parts like URLs, emails, IP addresses,
 
 127    DNS names (can be turned off). Replaced by a 'replaced-xxx' string
 
 128    followed by an XML tag containing the replaced text as attribute.
 
 129  - Acronyms like U.S.A. are systematically written with a final dot,
 
 130    even if it is missing in original file.
 
 131  
 
 132  
 
 133  In normal mode, all journal outputs are done via Python standard logging system,
 
 134  standard output is only used if a) you run the module in pipe mode (ie.
 
 135  results goes to stdout), or b) you set DEBUG or DEBUG_PREPROCESS global
 
 136  variables and you use the module directly on command line (which make journal
 
 137  and other traces to be sent to stdout).
 
 138  
 
 139  For an example of logging use, see :func:`enable_debugging_log` function.
